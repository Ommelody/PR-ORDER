# THAMC PR ORDER — คู่มือติดตั้ง (ระบบพร้อมใช้งานบน GitHub)

ระบบเสนอความต้องการซื้อ/จ้าง/เช่า — ดีไซน์ทางการ เลือกธีมสีได้ 6 โทน · เมนูยืดหด · รองรับมือถือเต็มรูปแบบ
สถาปัตยกรรม: **หน้าเว็บ (index.html) โฮสต์บน GitHub Pages** ⇄ **Backend บน Google Apps Script** ⇄ **Google Sheets + Drive**

```
GitHub Pages  ──(fetch JSON)──▶  Apps Script Web App  ──▶  Google Sheets (ทุกชีท) + Drive
   index.html                        Code.gs
```

> **เปิดดูได้ทันที:** ดับเบิลคลิก `index.html` เข้า **โหมดใช้งานในเครื่อง** (ข้อมูลบันทึกใน localStorage) ทดลองได้ครบทุกฟังก์ชัน · บัญชีทดลองรหัส `1234` (kamonwan / thanakorn / sawarin)

---

## ความสามารถหลัก
- **บทบาท (RBAC):** ผู้ขอซื้อ · เจ้าหน้าที่พัสดุ (คีย์คำขอเองได้ด้วย) · ผู้ดูแลระบบ
- **Workflow:** สร้างคำขอ → พัสดุรับเรื่อง (เลือกเงื่อนไข) → กำลังดำเนินการ → อนุมัติ PO → ตรวจรับ → เสร็จสิ้น
  - เงื่อนไขรับเรื่อง: **ไม่เกิน 500,000** → ดำเนินการ SAP · **เกิน 500,000** → SAP & EGP · **เครื่องมือแพทย์** → ส่งตรวจเอกสาร (เจ้าหน้าที่กดอัปเดตเอง)
- **ส่งกลับแก้ไข & ส่งซ้ำ:** เมื่อส่งเอกสารเดิมกลับมาอีกครั้ง ระบบสแตมป์ "รับครั้งที่ 2" อัตโนมัติ
- **เอกสาร A4:** ใบเสนอความต้องการซื้อ/จ้าง/เช่า พร้อมโลโก้ THAMC + QR Code เลขที่เอกสาร + พิมพ์ได้
- **Export Excel (SAP):** 19 คอลัมน์ตามรูปแบบ import SAP — **เฉพาะเจ้าหน้าที่/ผู้ดูแล**
- **Dashboard:** กราฟรายเดือน · โดนัทสัดส่วนประเภทพัสดุ · Top 5 หน่วยงาน · งานค้างตามขั้นตอน (คลิกเจาะข้อมูลได้)
- **ติดตามสถานะ:** ค้นหาบางส่วนของเลขเอกสารได้ + สแกน QR ด้วยกล้องมือถือ (เจ้าหน้าที่/แอดมินอัปเดตสถานะจากป็อปอัพได้ทันที)
- **ธีมสี 6 โทน:** เขียวมรกต · น้ำเงินกรมท่า · คราม · เขียวหัวเป็ด · แดงเลือดหมู · เทากราไฟต์

## ขั้นตอนที่ 1 — Backend บน Apps Script
1. เปิด https://script.google.com → **New project** → วางเนื้อหา `Code.gs`
2. `SHEET_ID` ตั้งค่าให้ตรงกับสเปรดชีตของคุณแล้ว (`1jzFYuLp03-…WW00w`)
   วางฟังก์ชันธุรกิจเดิม (submitPurchaseRequest, updatePurchaseRequestStatus ฯลฯ) ต่อท้ายได้
3. **Deploy → New deployment → Web app** · Execute as: **Me** · Who has access: **Anyone**
4. คัดลอกลิงก์ที่ลงท้าย `/exec`

## ขั้นตอนที่ 2 — ต่อหน้าเว็บกับ Backend
เปิด `index.html` แก้ `CONFIG` บนสุดของ `<script>`:
```js
const CONFIG = {
  WEB_APP_URL: "https://script.google.com/macros/s/XXXX/exec",  // ← วางลิงก์
  SHEET_ID: "1jzFYuLp03-gEtODbzr5U47zHBRHJVfXAWwj6O9WW00w"
};
```
เมื่อใส่ `WEB_APP_URL` ระบบจะเรียก `readAll` ดึงข้อมูล **จากทุกชีท** (Users, PurchaseRequests, ItemTypes, Departments, …) มาแสดงจริง และบันทึกกลับผ่าน Apps Script

## ขั้นตอนที่ 3 — ขึ้น GitHub Pages
1. สร้าง repository ใหม่ → อัป `index.html` (ไฟล์เดียว รวมทุกอย่างแล้ว รวมโลโก้)
2. **Settings → Pages → Source: main / root → Save**
3. ได้ลิงก์ `https://<user>.github.io/<repo>/` — เปิดใช้ได้ทั้งคอมพิวเตอร์และมือถือ (เพิ่มลง Home Screen ได้เหมือนแอป)

## หมายเหตุ
- **CORS:** หน้าเว็บส่งแบบ `text/plain` (simple request) — Apps Script จัดการ cross-origin ให้เอง ไม่ต้องตั้ง proxy
- **สแกน QR:** ใช้กล้องหลังของมือถือ ต้องเปิดผ่าน HTTPS (GitHub Pages เป็น HTTPS อยู่แล้ว) และอนุญาตสิทธิ์กล้องเมื่อระบบถาม
- **ล็อกอินด้วยใบหน้า/ลายนิ้วมือ (WebAuthn):** เข้าด้วยรหัสผ่านครั้งแรก → เมนูโปรไฟล์ → "เปิดใช้งานบนเครื่องนี้" (ใช้ Face ID/ลายนิ้วมือของเครื่อง ข้อมูลใบหน้าไม่ออกจากอุปกรณ์) ครั้งต่อไปกดปุ่มบนหน้าล็อกอินได้เลย · ผูกกับอุปกรณ์+บัญชี · ต้องเปิดผ่าน HTTPS · Code.gs เก็บ credential ในคอลัมน์ `BiometricCredIDs` (สร้างให้อัตโนมัติ)
- **ขออนุมัติงบประมาณ (BR):** โมดูลแยกจาก PR ใช้เลข `BR-YYYYMM-NNNN` — หน่วยงานกรอกคำขอ (ชื่อรายการ/เหตุผล/จำนวน/หน่วยนับ/มูลค่างาน/วันที่ต้องการ/เอกสารแนบ) → ผู้บริหารพิจารณา (อนุมัติ/ไม่อนุมัติ + หมายเหตุ) → แผนกงบประมาณตรวจสอบรหัสกับฐานข้อมูล (DataBase) — มีรหัสแล้วจัดสรรงบได้ทันที, ไม่มีรหัสต้อง "ส่งออกรหัส" ก่อน แล้วกรอกรหัสที่ได้รับ + กด "รับทราบ" → แจ้งเตือนผู้ขอทุกขั้น → ผู้ขอกด **Copy to PR** นำข้อมูลไปสร้างใบขอซื้อต่อได้ทันที
  - เมนู: "ขออนุมัติงบประมาณ" (ทุก role) · "พิจารณางบประมาณ" (Admin = ผู้บริหาร) · "จัดสรรงบประมาณ" (Staff/Admin = งบประมาณ/การเงิน)
  - จุดเชื่อมกับโปรแกรมออกรหัสภายนอก: Code.gs มีฟังก์ชัน `receiveItemCode(itemName, itemCode, brId)` เรียกผ่าน Web App action `"receiveItemCode"` — โปรแกรมออกรหัสของคุณ POST ข้อมูลนี้มาได้โดยตรงเพื่ออัปเดตสถานะอัตโนมัติ (ในตัวอย่างนี้มีปุ่ม "กรอกรหัสที่ได้รับ" ให้เจ้าหน้าที่กรอกเองเป็นทางเลือกสำรอง)
  - Export Excel to SAP สำหรับรายการที่จัดสรรงบแล้ว (เฉพาะเจ้าหน้าที่/แอดมิน)
  - **ข้อสมมติ:** ระบบมี 3 บทบาท (ผู้ขอซื้อ/เจ้าหน้าที่พัสดุ/ผู้ดูแลระบบ) — ให้ Admin ทำหน้าที่ "ผู้บริหาร" พิจารณางบ และ Staff ทำหน้าที่ "แผนกงบประมาณ" จัดสรรงบ หากต้องการแยกบทบาทจริงเพิ่ม แจ้งได้ครับ
- **เอกสารใบขออนุมัติงบประมาณ (A4):** สร้างอัตโนมัติเมื่อส่งคำขอ — มีโลโก้/QR เหมือนเอกสาร PR + ระบุปีงบประมาณ พิมพ์ได้ทันที
- **ไทม์ไลน์รวม BR→PR:** เมื่อคำขอ BR ถูกจัดสรรงบและ "Copy to PR" ไปสร้างใบขอซื้อแล้ว ทุกจุดติดตาม (ป็อปอัพ/หน้าติดตามสถานะ/รายงาน) จะแสดงขั้นตอนไหลต่อกันเป็นเส้นเดียว — ขั้นตอนฝั่งงบประมาณ (BR) เป็นจุดสีฟ้า ฿ ขั้นตอนฝั่งจัดซื้อ (PR) เป็นจุดสีเขียวตามเดิม พร้อมปุ่มเปิดเอกสารทั้งสองฝั่งได้จากจุดเดียว
- **ติดตามสถานะ + รายงาน รวม PR และ BR:** ค้นหาเลขที่เอกสารในหน้าติดตามสถานะรองรับทั้ง PR และ BR (พิมพ์บางส่วนก็ได้) ผลลัพธ์มีป้ายกำกับ PR/BR ให้แยกชัดเจน ส่วนหน้ารายงานมีการ์ดสรุปคำขออนุมัติงบประมาณแยกต่างหากต่อท้ายตารางรายงาน PR
- **เชื่อมกับระบบออกรหัสพัสดุ (Item Pro):** ในหน้าจัดสรรงบ ปุ่มสีเหลือง "ส่งออกรหัส (ค้นหาใน Item Pro)" ค้นรหัสที่ขึ้นต้นด้วย 8 จากฐานข้อมูลสินค้าที่ตรงชื่อรายการ — พบแล้วกดเลือกได้ทันที ไม่พบให้กด "แจ้งขอออกรหัสใหม่ที่ Item Pro" (ตั้งค่า `CONFIG.ITEM_PRO_URL` เพื่อเปิดลิงก์ระบบจริง)
  - **ข้อจำกัดที่ควรทราบ:** Item Pro เป็นระบบจริงที่มีฐานข้อมูลของตัวเอง (พบว่าเป็นระบบ Item Master Pro V2 ใช้ REST API) — เวอร์ชันนี้จำลองการค้นหาจากรายการโค้ด "8" ที่ซิงก์เข้ามาในฐานข้อมูลสินค้าของเราเอง (ชีต DataBase) เท่านั้น การเชื่อมสดจริงต้องให้ทีม Item Pro เปิด API/สิทธิ์อ่านข้อมูลให้ระบบนี้เรียกได้ — ถ้ามี endpoint หรือ Google Sheet ที่ใช้ร่วมกัน แจ้งมาได้ ผมจะต่อให้เป็นการเชื่อมสดจริง
- **ยังไม่ต่อ backend ก็ใช้ได้:** ถ้าเว้น `WEB_APP_URL` ว่าง ระบบทำงานเต็มรูปแบบด้วยข้อมูลในเครื่อง (เหมาะกับสาธิต/อบรม)
- **โลโก้:** ฝังในไฟล์ (base64) แล้ว แสดงบนหัวเอกสาร A4 โดยไม่ต้องพึ่งไฟล์ภายนอก
